# Deep Research

Use this skill when the user asks a question that benefits from current, verifiable information
from the web rather than memory alone — comparisons, "what's the latest on …", fact-checking,
gathering options, or any answer where a wrong detail matters.

## Method

1. **Frame the question.** Restate what is actually being asked and list the 2–4 sub-questions whose
   answers, together, settle it. If the request is ambiguous, pick the most useful interpretation and
   say which one you chose.

2. **Search broadly, then narrow.** Run several searches with *different* phrasings and angles — do
   not stop at the first result page. Prefer primary sources (official docs, the project/vendor itself,
   standards, original reporting) over aggregators and SEO content.

3. **Open and read, don't skim titles.** Fetch the promising sources and read the relevant sections.
   A snippet is a lead, not an answer.

4. **Cross-check every load-bearing claim.** A fact that decides the answer must appear in at least two
   independent sources, or be a primary source. If sources disagree, say so and explain which is more
   credible and why (recency, authority, methodology).

5. **Note recency.** State how current each key fact is; flag anything that may have changed since its
   source was published.

## Output

- Lead with the direct answer in 1–3 sentences.
- Then the supporting findings, each with its source (title + link).
- Call out uncertainty explicitly: what you could not verify, where sources conflicted, and what would
  resolve it.
- Never present an unverified claim as settled fact. If the web did not yield enough to answer, say
  that plainly instead of guessing.
