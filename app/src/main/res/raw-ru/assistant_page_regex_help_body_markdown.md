# Что делает этот редактор

Правила Regex переписывают текст в определённых точках конвейера чата.

Вы можете использовать их, чтобы:
- очищать импортированное форматирование SillyTavern
- нормализовать имена или пунктуацию перед отправкой в модель
- скрывать лишний текст только в UI
- применять ST-style размещения, такие как пользовательский ввод, вывод AI, world info, slash commands или reasoning

## Быстрый старт

1. Дайте правилу понятное имя.
2. Напишите шаблон в **Find Regex**.
3. Напишите результат в **Replace String**.
4. Решите, должно ли оно влиять на реальное сообщение, только на UI или только на prompt.
5. Если это правило пришло из ST, настройте **ST placements**, чтобы момент применения совпадал с исходным скриптом.

## Демонстрация 1: унификация вариантов ника

```text
Цель:
Преобразовать "Alicia" и "Ally" в "Alice"

Find Regex:
Alicia|Ally

Replace String:
Alice
```

## Демонстрация 2: очистка захваченной группы перед обратной записью

```text
Цель:
Сохранить только полезную часть совпадения

Find Regex:
Name:\s*(.*)

Replace String:
$1

Trim captured strings:
[
]
```

## Важные поля

- **Trim captured strings**: удаляет лишний обрамляющий текст из захваченных групп перед тем, как замена будет записана обратно.
- **RAW / ESCAPED macro substitution**: подставляет макросы, такие как `{{char}}` и `{{user}}`, прямо внутри самого regex перед сопоставлением.
- **ST placements**: если это поле не пустое, оно переопределяет общие области применения выше.
- **Run on edit**: также применяет правило при редактировании существующего сообщения.

> Практическое правило: если вы копируете regex из SillyTavern, сначала настройте ST placements. Если вы создаёте общее правило очистки для всего приложения, начните с общих областей применения.
