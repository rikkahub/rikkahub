# Что делают lorebooks

Lorebooks — это информация о мире, которая активируется по ключевым словам.

Каждый lorebook содержит множество записей. Каждая запись отслеживает недавний контекст и при совпадении добавляет своё содержимое в prompt.

## Подходящие сценарии использования

- факты о сеттинге
- заметки об отношениях
- сводки о фракциях или локациях
- повторяющиеся правила, которые должны появляться только когда это уместно

## Быстрый старт

1. Создайте lorebook.
2. Привяжите его к assistants, которые должны его использовать.
3. Добавьте записи с короткими, конкретными ключевыми словами.
4. Пишите лаконичное содержимое записей.
5. Протестируйте на реальных сообщениях чата, затем уточните ключевые слова, если запись срабатывает слишком часто.

## Пример

```text
Lorebook: Kingdom of Vale

Entry keywords:
vale, royal family, queen elira

Entry content:
Vale — прибрежное королевство под властью королевы Элиры. Королевская семья ставит дипломатию на первое место и прибегает к открытой войне только в крайнем случае.
```

## Настройки на уровне книги

- **Recursive scanning**: сработавшие записи могут помочь активировать дополнительные записи на следующих проходах.
- **Token budget**: ограничивает объём содержимого, который этот lorebook может добавить.
- **Assistant bindings**: позволяйте одному lorebook использоваться совместно только там, где ему место.

## Практические советы

- Лучше несколько небольших записей, чем одна огромная.
- Если одно ключевое слово слишком общее, добавьте вторичные ключевые слова или выборочную логику.
- Используйте **Constant Active** экономно.

> Если что-то должно появляться только тогда, когда контекст это упоминает, lorebook обычно является правильным инструментом.
