# Редактирование lorebook

Этот экран управляет одной книгой и списком её записей.

## Поля уровня книги

- **Название**: для организации.
- **Описание**: необязательная заметка для людей.
- **Включено**: включает или выключает всю книгу.
- **Рекурсивное сканирование**: позволяет уже сработавшим записям активировать больше записей при следующих проходах.
- **Бюджет токенов**: ограничивает объём текста из этой книги, который можно вставить.

## Рекомендуемый порядок работы

1. Создайте книгу.
2. Укажите, должна ли она быть рекурсивной.
3. Решите, нужен ли ей бюджет токенов.
4. Добавляйте записи по одной.
5. Проверяйте на реальных примерах чата.

## Демонстрационный сценарий

```text
Книга:
School Life

Сценарий использования:
Добавляйте детали кампуса только тогда, когда появляются темы занятий, клуба, общежития или экзаменов.

Рекомендуемая настройка:
- Рекурсивное сканирование: сначала выключено
- Бюджет токенов: сначала оставьте пустым
- Добавьте отдельные записи для общежития, студенческого совета, правил экзаменов и культуры клубов
```

## Когда менять рекурсивное сканирование

Включайте его только тогда, когда одна сработавшая запись должна иметь возможность пробуждать связанные записи при следующих проходах.

> Большинству книг стоит начинать с простого: рекурсивное сканирование выключено, без бюджета токенов, а затем при необходимости ужесточить настройки.
