# Lorebook 的作用

Lorebook 是由关键词触发的世界信息。

每个 Lorebook 都包含许多条目。每个条目都会监视最近的上下文，并在匹配时将其内容注入到提示词中。

## 适用场景

- 设定事实
- 关系备注
- 派系或地点摘要
- 仅在相关时才应出现的重复规则

## 快速开始

1. 创建一个 Lorebook。
2. 将它绑定到应使用它的助手。
3. 添加带有简短且具体关键词的条目。
4. 编写简洁的条目内容。
5. 使用真实聊天消息进行测试；如果触发过于频繁，就收紧关键词。

## 示例

```text
Lorebook: Kingdom of Vale

条目关键词：
vale, royal family, queen elira

条目内容：
Vale 是一个沿海王国，由 Queen Elira 统治。王室重视优先通过外交解决问题，只有在万不得已时才会发动公开战争。
```

## 书籍级设置

- **递归扫描**：已触发的条目可能会在后续扫描中帮助触发更多条目。
- **Token 预算**：限制此 Lorebook 最多可贡献多少内容。
- **助手绑定**：让一个 Lorebook 只在适用的地方共享。

## 实用建议

- 与其写一个巨大的条目，不如使用多个小条目。
- 如果某个关键词过于宽泛，可添加次级关键词或选择性逻辑。
- 谨慎使用 **Constant Active**。

> 如果某些内容只应在上下文提到时出现，Lorebook 通常就是合适的工具。
