# 编辑单个 Lorebook 条目

这里用于定义条目**何时**触发以及**注入什么**。

## 推荐工作流程

1. 编写主要关键词。
2. 添加内容。
3. 选择注入位置。
4. 在聊天中测试。
5. 完成以上步骤后，再添加概率、分组、sticky、cooldown 或 recursion 标记等高级控制项。

## 示例 1：简单关键词条目

```text
主要关键词：
lighthouse, harbor beacon

内容：
Old Harbor 的 lighthouse 已废弃 20 年，但当地人仍相信它的灯会在暴风雨来临前亮起。
```

## 示例 2：让宽泛触发更安全

```text
主要关键词：
king

次要关键词：
vale, elira

选择逻辑：
任一次要关键词
```

这意味着，该条目不会仅因为有人在随机语境中说了“king”就被触发。

## 重要部分

- **触发条件**：关键词、regex 模式、大小写敏感、整词匹配、概率以及额外触发来源。
- **注入设置**：内容插入的位置以及使用的角色。
- **高级兼容性**：ST 风格的 group、delay、sticky、cooldown、recursion 和导出元数据。

## 实用建议

- 保持内容真实且简洁。
- 优先使用精确关键词，而不是写超长的条目内容。
- 仅在你确实需要类似 ST 的运行时行为时，才使用高级兼容性。

> 如果你不确定，请先不要调整高级设置，先让基础触发正常工作。
