# 로어북 항목 하나 편집하기

여기서 항목이 **언제** 트리거되는지와 **무엇을** 삽입하는지 정의합니다.

## 권장 작업 순서

1. 기본 키워드를 작성합니다.
2. 내용을 추가합니다.
3. 삽입 위치를 선택합니다.
4. 채팅에서 테스트합니다.
5. 그다음에야 확률, 그룹, sticky, cooldown 또는 recursion 플래그 같은 고급 제어를 추가합니다.

## 데모 1: 간단한 키워드 항목

```text
기본 키워드:
lighthouse, harbor beacon

내용:
The Old Harbor lighthouse는 20년 동안 버려져 있었지만, 현지인들은 폭풍이 오기 전에 등불이 켜진다고 아직도 믿고 있습니다.
```

## 데모 2: 넓은 트리거를 더 안전하게 만들기

```text
기본 키워드:
king

보조 키워드:
vale, elira

선택 로직:
보조 키워드 중 아무거나
```

이는 누군가가 무작위 맥락에서 "king"이라고 말했다고 해서 항목이 실행되지 않는다는 뜻입니다.

## 중요한 섹션

- **트리거 조건**: 키워드, regex 모드, 대소문자 구분, 단어 전체 일치, 확률, 추가 트리거 소스.
- **삽입 설정**: 내용이 들어갈 위치와 사용할 역할.
- **고급 호환성**: ST 스타일 그룹, 지연, sticky, cooldown, recursion 및 내보내기 메타데이터.

## 실용적인 조언

- 내용은 사실에 기반하고 간결하게 유지하세요.
- 긴 항목 내용보다 정확한 키워드를 우선하세요.
- ST와 유사한 런타임 동작이 실제로 필요할 때만 고급 호환성을 사용하세요.

> 확실하지 않다면 고급 설정은 건드리지 말고 먼저 기본 트리거가 작동하도록 하세요.
