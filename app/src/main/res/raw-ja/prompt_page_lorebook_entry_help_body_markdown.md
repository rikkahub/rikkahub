# 1つのロアブック項目を編集する

ここでは、項目が**いつ**発動し、**何を**挿入するかを定義します。

## 推奨ワークフロー

1. 主要キーワードを書く。
2. コンテンツを追加する。
3. 挿入位置を選ぶ。
4. チャットでテストする。
5. その後で初めて、確率、グループ、sticky、cooldown、または recursion フラグのような高度な設定を追加する。

## デモ1: シンプルなキーワード項目

```text
Primary keywords:
lighthouse, harbor beacon

Content:
Old Harbor の灯台は20年前に廃墟となったが、地元の人々はいまでも嵐の前にその灯りがともると信じている。
```

## デモ2: 広すぎるトリガーをより安全にする

```text
Primary keyword:
king

Secondary keywords:
vale, elira

Selective logic:
Any secondary keyword
```

これは、誰かが無関係な文脈で "king" と言っただけでは、この項目が発動しないことを意味します。

## 重要なセクション

- **トリガー条件**: キーワード、regex モード、大文字・小文字の区別、単語全体の一致、確率、および追加のトリガーソース。
- **挿入設定**: コンテンツの挿入先と、使用するロール。
- **高度な互換性**: ST スタイルのグループ、delay、sticky、cooldown、recursion、およびエクスポートメタデータ。

## 実用的なアドバイス

- コンテンツは事実に基づき、簡潔に保つ。
- 長大な項目コンテンツよりも、正確なキーワードを優先する。
- 高度な互換性は、実際に ST のようなランタイム動作が必要な場合にのみ使う。

> よく分からない場合は、高度な設定には触れず、まず基本的なトリガーを機能させてください。
