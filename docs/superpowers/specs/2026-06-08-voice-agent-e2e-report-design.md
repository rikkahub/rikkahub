# Voice Agent E2E Report Design

## Goal

Make the live Voice Agent Hermes/Gbrain E2E easy to understand by starting from one known text prompt, generating PCM
from that text, running the real voice pipeline, and producing a local report that shows what happened end to end.

## Problem

The current E2E proves that the runtime pipeline can reach Gemini, call `ask_hermes`, call Hermes/Gbrain, receive output
audio, and play audio. It is still hard to interpret because the operator cannot see the full conversation in one place.
The PCM may come from a previous local file, Gemini's understood transcript may not be preserved reliably in the
conversation database, and the raw Hermes answer is saved separately from the source prompt and transcript.

## Recommended Flow

The script owns a configurable source prompt:

```bash
VOICE_AGENT_E2E_PROMPT_TEXT="Please ask Hermes if he is connected to G-Brain. Please answer with yes or no."
```

If `VOICE_AGENT_E2E_PCM_PATH` is set, the script keeps the existing behavior and injects that PCM. If it is not set, the
script generates signed 16-bit little-endian mono PCM at 16 kHz from `VOICE_AGENT_E2E_PROMPT_TEXT` and writes it to:

```text
build/voice-agent-e2e/generated-prompt.pcm
```

The generated or supplied PCM is injected through the existing debug audio hook, so the app still exercises the real
Voice Agent capture path, Gemini Live, `ask_hermes`, Hermes/Gbrain, Gemini tool response handling, output audio, and
Android playback.

## App-Private Artifacts

During manual-review live runs, the app writes these app-private E2E artifacts under `no_backup/voice-e2e/`:

```text
input-transcript.txt
hermes-call.txt
hermes-answer.txt
output-transcript.txt
```

The app writes these files as trace-keyed Voice Agent artifacts for every session. These artifacts are not logged to
logcat by default. The runner clears stale copies before a run and removes app-private copies during cleanup.

The source text used to generate PCM stays in the local runner artifact `build/voice-agent-e2e/generated-prompt.txt` and
is not copied into app-private storage. If an operator supplies an existing PCM, the report records `missing` unless
`VOICE_AGENT_E2E_PROMPT_TEXT` is also provided for that explicit PCM run.

`input-transcript.txt` is what Gemini transcribed from the injected voice. `hermes-call.txt` is the raw prompt Gemini
sent in the `ask_hermes` tool call. `hermes-answer.txt` is Hermes's raw answer. `output-transcript.txt` is Gemini's
spoken response text to the user, when Gemini provides output transcription.

## Local Report

After all live markers pass in manual-review mode, the runner pulls the app-private artifacts with `run-as` and writes:

```text
build/voice-agent-e2e/report.txt
```

The report contains six sections in this order:

- `Text used to generate voice`: the exact source text when known, otherwise `missing`.
- `Gemini understood from voice`: Gemini's input transcription text, or `missing` if Gemini did not emit one.
- `Hermes call`: the raw `ask_hermes` prompt generated by Gemini.
- `Hermes elapsed time`: the app-observed elapsed time and server elapsed time parsed from the safe hash marker.
- `Hermes answer`: Hermes's raw answer.
- `Gemini response to user`: Gemini's output transcription text, or `missing` if Gemini did not emit one.

The script prints the report path, not the raw report content. The operator reads the local report and decides whether
the run passed semantically.

## Pass Criteria

The script still requires these runtime markers:

- ADB device ready.
- Target app installed.
- Gemini setup complete.
- PCM injected into the app debug audio capture path.
- Gemini emits `ask_hermes`.
- Hermes response hash emitted, including elapsed timing.
- Gemini tool response sent.
- Gemini output audio received.
- Android playback queued and wrote audio.
- Local report generated.

Manual semantic pass requires the report to show a coherent end-to-end conversation. For the default prompt, the Hermes
answer should clearly answer whether Hermes is connected to Gbrain with yes/no meaning.

## Privacy

Raw prompt text, transcripts, Hermes call text, Hermes answer, PCM files, local reports, and logcat artifacts are local
only and must not be committed or pasted into shared logs. Logcat should keep safe hash/timing markers by default.

## Future Extension

The script can later support optional semantic checks:

```bash
VOICE_AGENT_E2E_EXPECT_INPUT_TRANSCRIPT_REGEX='ask Hermes'
VOICE_AGENT_E2E_EXPECT_HERMES_ANSWER_REGEX='^(yes|no)\b'
VOICE_AGENT_E2E_EXPECT_OUTPUT_TRANSCRIPT_REGEX='^(yes|no)\b'
```

These are optional and should not block the first report-focused implementation.
